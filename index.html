HTML

  • HTML stands for HyperText Markup Language
  • Child elements are elements that are nested in other elements
  • Parent elements are elements that holds all of the child elements
  • "head" elements contain information about the webpage
  • "body" elements represent the visible content shown to the user
  • "style" elements define the CSS styles associated with a website
  • "base" elements define the base url for a webpage
  • "link" elements connect an external resource to the HTML document
  • "meta" elements define metadata such as the character set, description, keywords, author, and viewport
  • "script" elements embed client-side scripts in the HTML document

CSS

  • CSS stands for Cascading Style Sheets
  • Incline CSS is writen directly into an HTML file
  • The Internal CSS style sheet contains CSS rules for the webpage in the head section of the HTML file
  • The External CSS style sheet is the most common way to style a webpage, it's also prefferred in most cases because it allows developers to keep all of their CSS rules in a seperate file
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Declarations contain two important components; the CSS property we want to apply, and the value of the property
  • The "*" symbol applies CSS to all of the elements visible on a webpage

Git

  • GitHub is the web interface for git, a verson contol software that allows multiple people to make seperate changes to a program at the same time without getting in each others way
  • A repository or repo, is a folder for your project that contains all of the files and their revision history
  • A branch is a workplace where you can make changes that don't affect the live site
  • Command "git checkout -b branch-name" creates a new branch and switches to it
  • Command "git status" checks what branch we are currently on
  • Whenever you add, edit, or delete a file you're making a commit (commits add changes to your branch)
  • A pull request or pr is when you're asking for changes on a branch to merge into another branch
  • An issue is a great way to report new tasks or content fixes and track your progress
  • Forking is when you copy another user's repository to your own account, and then you can make any changes you want to your copy without affecting the original
  • Versioning refers to recording and tracking the revisions made to a project
